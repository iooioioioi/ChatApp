# ⚡ SNABB DOKUMENTATIONS-REFERENS

## 6 Filer Du Behöver Skapa

### 1️⃣ **README.md** (i roten)
```
- Projektnamn & beskrivning (1-2 meningar)
- Systemkrav (.NET version)
- Snabbstart (2 commands)
- Mappstruktur
- Teknologier (C#, WPF, TCP)
```

### 2️⃣ **docs/ClassDiagram.md** 
```
- Alla 7 klasser med properties & metoder:
  • Message (Sender, Content, Timestamp, Serialize())
  • User (Username, Id, ConnectedTime)
  • Logger (Log(), LogError())
  • ChatServer (_listener, _clients, Start(), HandleClient(), BroadcastMessage())
  • ClientConnection (_tcpClient, User, ReadMessage(), SendMessage())
  • ServerConnection (ConnectAsync(), SendMessage(), Disconnect())
  • MainWindow (Connect, Send, MessageHistory display)
  
- Relationer mellan klasser
- Namespace-indelning
```

### 3️⃣ **docs/Design.md**
```
- Arkitektur-diagram (Server ↔ Klient)
- Nätwerks-protocol:
  SENDER|CONTENT|TIMESTAMP (pipe-separerat)
  
- 4 Designmönster:
  • Threading - varje klient i egen tråd
  • Async/Await - responsiv UI
  • Observer Pattern - events för meddelanden
  • Dependency Injection - Logger
  
- Thread-safety (lock på _clients lista)
```

### 4️⃣ **docs/UserManual.md**
```
- Systemkrav
- Installation (3 steg)
- Start Server (1 command)
- Start Klient (1 command)
- GUI-beskrivning & layout
- Hur man använder (anslut → chatta → koppla ifrån)
- 3-5 vanliga fel + lösningar
- Exempel session
```

### 5️⃣ **docs/TestPlan.md**
```
- Minst 8-10 testfall:
  ✓ Serverstart
  ✓ Klientstart
  ✓ Anslutning
  ✓ Enkelt meddelande
  ✓ Flera klienter
  ✓ Broadcast
  ✓ Frånkoppling
  ✓ Loggning
  
- För varje test:
  • Vad du gör
  • Förväntat resultat
  • Faktiskt resultat (✓/✗)
  
- Stress-test (många meddelanden)
```

### 6️⃣ **docs/Evaluation.md**
```
**Vilka funktioner fungerar?**
- ✓ Server hanterar flera klienter
- ✓ Broadcast
- ✓ GUI
- ✓ Loggning
- Etc.

**Prestanda:**
- Meddelandelatens: ~50-100ms
- Minneskonsumption: ~1MB per klient
- Max klienter testade: X
- CPU-användning: låg

**Kodkvalitet:**
- OOP-koncept demonstrerade? ✓
- Thread-safe? ✓
- Felhantering? ✓
- Läsbar kod? ✓

**Förbättringar som kunde göras:**
- Privatmeddelanden
- User-list
- Kryptering
```
